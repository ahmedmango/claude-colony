CLAUDE COLONY

watch your AI agents work, without watching logs
◉ How it saves your laptop. Colony is a thin viewer. The actual Claude agents run on a remote worker (fly.io, lambda, or your own VPS). Your browser just gets a tiny WebSocket stream of state updates to draw ants with. No agent SDK on your machine. Fans stay quiet.
▸ your laptop ──(websocket state)── remote worker ──(agent sdk)── claude api

◆ HOW TO READ THE COLONY

🐜
Ants are agents. Each ant is one Claude agent. Watch it walk through your filesystem — the tunnels literally map to your repo directories.
🎨
Color tells you what they do. Red scouts read, black workers edit, grey soldiers run tests / review. Glowing dot on their back = code diff they're carrying.
💬
Speech bubble = agent needs you. You cannot miss it: big glowing bubble + pulsing orange ring around the ant + ⚠ NEEDS YOU inbox glows red top-right + browser tab title flashes. Click bubble or inbox item to answer.
♕
You are the queen. Bottom center. Eggs = pending tasks waiting to be picked up. Drop a leaf on the surface = queue new work.
✨
Glowing chambers = events. Green = tests pass. Red = merge conflict or error. Orange pheromone trails = hot files being worked on right now.

◆ WHY YOUR LAPTOP STAYS COOL

Classic "run agents locally" burns CPU, RAM, and battery. Colony runs on this model instead:
your browser ◂ ─ ws ─ ▸ remote worker ◂ ─ ─ ▸ anthropic api
Local: render ant sprites, draw SVG tunnels. ~2% CPU.
Remote: agent SDK, filesystem ops, tool calls, git worktrees.
You can close your laptop. Ants keep digging. Reopen → same state. Works from phone.

◆ COLOR KEY

●
Red ant = scout. Reads files, gathers context. Cheap, fast.
●
Black ant = worker. Writes + edits. Carries cargo (diff bytes).
●
Grey ant (bigger) = soldier. Runs tests, reviews PRs, caretaker.
🟡
Yellow cargo = markdown/docs. Green = code. Blue = data/SQL.
🟢
Green chamber pulse = success (tests pass, file saved).
🔴
Red chamber pulse = failure (error, conflict).
